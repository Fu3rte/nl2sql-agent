# NL2SQL Agent — 需求文档

> 一个将自然语言自动转换为 SQL 并执行查询的单 Agent 应用。

---

## 项目目的

面向 Agent 开发的入门练手项目。核心目标是**熟悉单 Agent 的架构模式、Tool Calling 机制和 LangGraph 状态机**，而非追求产品完整度。前后端分离，前端极简化，重点在后端 Agent 逻辑。

---

## 功能列表

### F1 自然语言查询

用户输入自然语言问题，Agent 自动完成以下流程并返回结果：

1. 理解用户意图
2. 生成对应 SQL 语句
3. 执行 SQL
4. 将查询结果翻译为自然语言回答

**输入示例**：
- "上个月销售额最高的三个产品是什么"
- "哪些客户的订单总数超过 10 单"
- "退货率最高的品类是哪个"

### F2 SQL 与查询结果可视化

前端展示 Agent 每一步的产出，让用户看到 Agent 的思考过程：

- 用户原始问题
- Agent 生成的 SQL 语句（高亮显示）
- SQL 执行结果（表格展示）
- 自然语言回答

### F3 SQL 错误自修复

当生成的 SQL 执行失败时，Agent 自动将错误信息带回 LLM，重新生成 SQL 并重试，直到成功或达到重试上限。

- 最多重试 3 次
- 每次重试携带上一次的错误信息
- 超过重试上限后返回友好的错误提示

### F4 问题边界判断

Agent 能判断用户的问题是否需要数据查询：

- 数据查询类问题（"销售额多少"）→ 进入 SQL 生成流程
- 闲聊类问题（"你好"、"今天天气怎么样"）→ 直接由 LLM 回答

### F5 流式输出

Agent 的每一步状态变化通过 SSE 实时推送到前端，用户可以看到 Agent 正在"写 SQL → 执行中 → 翻译结果"的全过程，而非等待一次性返回。

### F6 示例数据库

内置一个电商场景的 SQLite 示例数据库，包含以下表结构：

- `customers` — 用户信息
- `products` — 商品信息
- `orders` — 订单记录
- `order_items` — 订单明细

用户启动即用，无需配置外部数据库。

### F7 自定义数据库接入（可选）

允许用户上传 SQLite 文件或输入连接信息，Agent 自动读取表结构并适配查询。

---

## 不做的功能

- 不做对话历史管理（本次练手焦点不在会话）
- 不做用户认证
- 不做多数据库同时查询
- 不做可视化图表生成
- 不做生产级部署配置

---

## 技术约束

- 后端：Python + FastAPI + LangGraph + LangChain
- 前端：React + TypeScript，极简 UI
- 数据库：本地 SQLite
- LLM：DeepSeek V4（Anthropic API 兼容格式）
